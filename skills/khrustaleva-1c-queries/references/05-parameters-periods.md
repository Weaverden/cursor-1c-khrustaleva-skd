# Параметры запроса в СКД и периоды

Источник: примеры 2, 7 (стр. 50, 81–86); пример 10 (стр. 112+).

## Автопараметры платформы

При использовании **виртуальных таблиц** платформа может создать `НачалоПериода`, `КонецПериода` автоматически.

При **объектных таблицах** (движения регистра, документы) — период в запрос **задаётся вручную**:

```bsl
ГДЕ
    СобранныеСБ.Период >= &НачалоПериода
    И СобранныеСБ.Период <= &КонецПериода
```

## Конец периода «включительно»

В настройках параметра **КонецПериода** — выражение:

```bsl
КонецПериода(&КонецПериода, "День")
```

И **Использование** = `Всегда`, если параметр обязателен в запросе — иначе ошибка при пустых значениях.

## Параметры, созданные из запроса

- `ВидДвижения` — из условия по регистру; часто скрыт от пользователя, значение `Приход`.
- `Ссылка` — параметр **связи** наборов; обычно недоступен пользователю.

## Параметры виртуальных таблиц

Предпочтительно передавать отбор в **параметры виртуальной таблицы**, а не только в `ГДЕ` после JOIN — см. также `composing-1c-queries` (общие правила 1С).

Пример остатков по периодам в СКД:

```bsl
ВЫБРАТЬ
    ОстаткиНоменклатурыОстаткиИОбороты.Период,
    ОстаткиНоменклатурыОстаткиИОбороты.Номенклатура,
    ОстаткиНоменклатурыОстаткиИОбороты.КоличествоНачальныйОстаток,
    ОстаткиНоменклатурыОстаткиИОбороты.КоличествоКонечныйОстаток
ИЗ
    РегистрНакопления.ОстаткиНоменклатуры.ОстаткиИОбороты(&НачалоПериода, &КонецПериода, , , ) КАК ОстаткиНоменклатурыОстаткиИОбороты
```

## Связь наборов по периодам

Два набора (остатки по периодам + продажи за период) связываются не только по номенклатуре, но и по **полю периода** — отдельный тип связи в СКД (пример 10).

## Обязательность параметров

Если параметр всегда передаётся в запрос — включить **обязательное использование**, чтобы отчёт формировался без ошибки при значениях по умолчанию.
