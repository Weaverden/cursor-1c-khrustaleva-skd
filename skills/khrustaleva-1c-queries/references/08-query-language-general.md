# Язык запросов 1С — общие правила (по Хрусталевой)

Источник: примеры гл. 2 (стр. 64–86, 112+) и справочник гл. 3. Применимо **везде**: модули, консоль, СКД, отчёты, обработки.

> Синтаксис и оптимизация платформы — в `composing-1c-queries`. Здесь — практические паттерны из книги.

## Соединения таблиц

### LEFT JOIN: «все слева, справа по совпадению»

Классический случай: **весь справочник** + остатки/цены **где есть**.

```bsl
ИЗ
    Справочник.Номенклатура КАК спрНоменклатура
        ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ОстаткиНоменклатуры.Остатки КАК Ост
        ПО Ост.Номенклатура = спрНоменклатура.Ссылка
```

В конструкторе запроса: флаг **«Все»** на **левой** таблице; на правой — снят. Иначе получится INNER — пропадут элементы без движений.

### INNER vs LEFT

- Нужны **только** пары «есть и слева, и справа» → INNER (или LEFT + отбор, эквивалентно при определённых условиях).
- Нужны **все** строки левой таблицы → LEFT, левая таблица в `ИЗ` первая.

### Несколько таблиц в одном запросе

- Осмысленные **алиасы таблиц** (`спрНоменклатура`, не `Т1`).
- **Псевдонимы полей** (`КАК Остаток`, `КАК Номенклатура`) — сразу понятный результат.
- Связи задавать явно в `ПО ...`; не полагаться на неявные соединения.

### Когда разбивать на несколько запросов / ВТ

Один тяжёлый JOIN не всегда лучший вариант. Разбивать, если:

- запрос нечитаем и дублирует строки;
- нужны промежуточные отборы больших множеств;
- дальше соединяется с данными другой природы (см. `ПОМЕСТИТЬ` в `composing-1c-queries`).

В **СКД** альтернатива — несколько **наборов данных** со связью (см. `03-dataset-links.md`).

## Виртуальные таблицы регистров

### Условия — в параметрах ВТ, не только в ГДЕ

```bsl
// Предпочтительно
РегистрНакопления.ОстаткиНоменклатуры.Остатки(
    &Дата,
    Номенклатура = &Номенклатура) КАК Ост

// Хуже: отбор только после JOIN в ГДЕ — лишняя работа
```

Для **ОстаткиИОбороты**, **СрезПоследних**, **СрезПервых** — период и измерения в параметрах виртуальной таблицы.

### Суффиксы полей виртуальных таблиц

| ВТ | Пример поля |
|----|-------------|
| `.Остатки` | `КоличествоОстаток` |
| `.Обороты` | `КоличествоОборот` |
| `.ОстаткиИОбороты` | `КоличествоНачальныйОстаток`, `Приход`, `Расход`, `КонечныйОстаток` |

### Остатки с детализацией по регистратору

При выборе `Регистратор` из `ОстаткиИОбороты` включать **родительское поле периода** (`ПериодСекунда` и т.д.) — иначе при агрегации в отчёте/СКД остатки считаются неверно (см. `06-balance-totals.md`).

## Параметры запроса `&Имя`

- Внешние значения — **только через параметры**, не конкатенация в текст.
- Период по **объектной таблице** регистра (не ВТ):

```bsl
ГДЕ
    Движения.Период >= &НачалоПериода
    И Движения.Период <= &КонецПериода
```

- Последний день периода включительно (в настройках отчёта/перед выполнением):

```bsl
КонецПериода(&КонецПериода, "День")
```

- Отбор по ссылке из другого запроса / родительской строки:

```bsl
ГДЕ Документ.Ссылка В (&Ссылка)
```

Используется в модулях и в **СКД** как параметр связи наборов.

## ГДЕ, отбор, сортировка, группировка

| Что | Рекомендация (книга) |
|-----|----------------------|
| Отбор, строки точно не нужны | `ГДЕ` в **запросе** |
| Сортировка для гибкости отчёта | настройки **СКД** (в модуле — `УПОРЯДОЧИТЬ ПО` в запросе) |
| Группировки отчёта | настройки **СКД** (в модуле — `СГРУППИРОВАТЬ ПО` в запросе) |
| Исключить группы справочника | `ЭтоГруппа = ЛОЖЬ` в `ГДЕ` |
| Вид движения регистра | `ВидДвижения = ЗНАЧЕНИЕ(...Приход)` в `ГДЕ` |

В **обычном коде** сортировку и группировку обычно задают в самом запросе — совет книги про разделение «жёсткий отбор в запросе / гибкая структура в СКД».

## ОБЪЕДИНИТЬ в языке запросов vs «Объединение» в СКД

| Контекст | Смысл |
|----------|--------|
| `ОБЪЕДИНИТЬ [ВСЕ]` в **одном** пакете запросов | Строки из двух `ВЫБРАТЬ` подряд, поля выровнены псевдонимами |
| Набор данных **«Объединение»** в **СКД** | Два отдельных запроса в схеме, складываются при компоновке |

Не путать с **связью** наборов / `ЛЕВОЕ СОЕДИНЕНИЕ` по ключу.

## Связь «запрос ↔ запрос» в коде (аналог СКД)

Паттерн из прим. 7: сначала выборка из регистра, затем детализация документов **только для нужных ссылок**:

```bsl
// Шаг 1: получить регистраторы
// Шаг 2:
ВЫБРАТЬ ... ИЗ Документ.СборкаКомпьютера ГДЕ Ссылка В (&МассивСсылок)
```

В СКД то же через `В (&Ссылка)` и параметр связи без ручного массива в коде.

## Чеклист для любого запроса

- [ ] ВТ: условия в параметрах таблицы
- [ ] LEFT: «все» слева, если нужен полный левый набор
- [ ] Понятные алиасы таблиц и полей
- [ ] Период и ссылки — через `&Параметры`
- [ ] Не путать UNION (все строки обоих наборов) и JOIN по ключу
- [ ] При остатках + регистратор — поля периодов из ВТ
