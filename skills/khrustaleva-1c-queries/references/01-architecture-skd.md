# Устройство СКД и наборы данных (гл. 1)

Источник: Хрусталева, гл. 1, стр. 9–26.

## Элементы схемы компоновки данных

1. **Наборы данных** — источники (запрос, объект, объединение).
2. **Связи наборов данных** — как связать несколько наборов (аналог LEFT JOIN между наборами).
3. **Вычисляемые поля** — поля отчёта поверх наборов.
4. **Ресурсы** — агрегируемые поля (сумма, количество…).
5. **Параметры** — в т.ч. параметры запросов и виртуальных таблиц.
6. **Настройки** — структура, отбор, группировки.

## Виды наборов данных

| Вид | Назначение |
|-----|------------|
| **Запрос** | Текст на языке запросов 1С к ИБ |
| **Объект** | Внешний набор (таблица значений и т.п.), заполняется из BSL |
| **Объединение** | Несколько подчинённых наборов; данные **не связываются**, а **складываются** (аналог `ОБЪЕДИНИТЬ ВСЕ`) |

## Связь vs объединение (критично)

- **Связь наборов** — левое внешнее соединение: все строки «родителя», из «приёмника» только совпадающие по ключу.
- **Объединение** — все строки из всех подчинённых наборов подряд (разная структура полей выравнивается псевдонимами).

Если нужны **все** записи из обоих источников без сопоставления по ключу — **Объединение**, не связь.

## Параметры и виртуальные таблицы

Параметры могут быть:

- явными в запросе (`&ВидНоменклатуры`);
- параметрами виртуальных таблиц (`Остатки(&Дата, Склад = &Склад)`).

Отбор **лучше в запросе**, если отсечённые строки точно не нужны в отчёте. Сортировку и группировку **лучше в настройках СКД** — гибче для пользователя.

## Имена полей

Имена полей в запросе (псевдонимы `КАК`) переносятся в схему сразу в **Поле / Путь / Заголовок** — задавать понятные имена в запросе, а не только в СКД.
