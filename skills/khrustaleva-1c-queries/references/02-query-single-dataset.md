# Один набор данных — запрос с несколькими таблицами

Источник: пример 4, стр. 64–71; советы стр. 88–93.

## Когда один запрос

Объединять таблицы **в одном запросе**, когда:

- нужен простой отчёт «справочник + остатки»;
- демонстрируется классическое соединение в языке запросов;
- не требуется раздельная компоновка наборов.

**Альтернатива:** два связанных набора данных (справочник + виртуальная таблица остатков) — предпочтительнее, если один запрос станет слишком сложным, нужны итоги без дублей, или смешиваются типы наборов (запрос + таблица значений).

## Паттерн: справочник + остатки (LEFT JOIN)

Задача: список номенклатуры **всех видов**, остатки — где есть.

```bsl
ВЫБРАТЬ
    спрНоменклатура.Ссылка КАК Номенклатура,
    спрНоменклатура.Родитель КАК Группа,
    спрНоменклатура.ВидНоменклатуры КАК Вид,
    ОстаткиНоменклатурыОстатки.Склад,
    ОстаткиНоменклатурыОстатки.КоличествоОстаток КАК Остаток
ИЗ
    Справочник.Номенклатура КАК спрНоменклатура
        ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ОстаткиНоменклатуры.Остатки КАК ОстаткиНоменклатурыОстатки
        ПО ОстаткиНоменклатурыОстатки.Номенклатура = спрНоменклатура.Ссылка
ГДЕ
    спрНоменклатура.ЭтоГруппа = ЛОЖЬ
```

### Настройка связи в конструкторе запроса

- Таблица **слева** (справочник): флаг **«Все»** — все строки левой таблицы попадают в результат.
- Таблица **справа** (остатки): флаг **«Все»** снят — левое соединение.

Иначе: в отчёт не попадёт номенклатура без движений по регистру.

## Отбор, сортировка, группировка

| Механизм | Где лучше |
|----------|-----------|
| Отбор, отсекающий строки навсегда | **В запросе** (`ГДЕ`) |
| Сортировка, группировки отчёта | **В настройках СКД** |

Пример отбора групп справочника: `спрНоменклатура.ЭтоГруппа = ЛОЖЬ`.

## Псевдонимы полей

Менять имена в запросе (`КАК Номенклатура`, `КАК Остаток`), чтобы в схеме СКД сразу заполнились поле, путь и заголовок.

## Переименование таблиц в запросе

При нескольких таблицах — осмысленные алиасы (`спрНоменклатура`, не однобуквенные), чтобы условия связи и `ГДЕ` читались однозначно.
