# Связи наборов данных в СКД

Источник: примеры 3, 7 (стр. 58–60, 81–84); справочник стр. 325–329.

## Модель связи

- **Источник (родитель)** — набор, от которого идёт связь.
- **Приёмник (зависимый)** — набор, к которому идёт связь.
- **Выражения** — поля-ключи (например `Регистратор` = `Ссылка`).
- **Параметр связи** — параметр запроса приёмника (`&Ссылка`), куда СКД подставляет значения из родителя.

По умолчанию все связи в СКД — **левые внешние**: все строки родителя; из приёмника — только совпадающие.

**Исключение:** глобальный отбор на поле **зависимого** набора → связи становятся **внутренними** (только совпадающие пары).

## Параметр связи в запросе приёмника (обязательно для производительности)

Без параметра связи дочерний набор выбирает **все** документы, СКД отбрасывает лишнее при компоновке.

```bsl
ВЫБРАТЬ
    СборкаКомпьютера.Дата,
    СборкаКомпьютера.Номер,
    СборкаКомпьютера.Ссылка
ИЗ
    Документ.СборкаКомпьютера КАК СборкаКомпьютера
ГДЕ
    СборкаКомпьютера.Ссылка В (&Ссылка)
```

В схеме: связь `Регистратор` (родитель) → `Ссылка` (приёмник), параметр **`Ссылка`**.

## Пример: срез цен + документ поступления

1. Набор **Цены** — `РегистрСведений.Цены.СрезПоследних`, поля `Номенклатура`, `Цена`, `Регистратор`.
2. Набор **Поставщики** — `Документ.ПриходнаяНакладная`, поля `Ссылка`, `Номер`, `Дата`, `Контрагент`.
3. Связь: `Регистратор` = `Ссылка`, родитель — **Цены** (все цены; документ — если есть).

Направление родителя меняет результат: родитель «Поступление» vs «Расход» — в отчёте разный «главный» набор (стр. 326–328).

## Правила (справочник, стр. 325–329)

1. Родитель / приёмник задают, чьи строки «главные».
2. Тип связи в схеме не выбирается — всегда LEFT, кроме глобального отбора на приёмнике.
3. **Параметр связи** в запросе приёмника — меньше лишних данных.
4. При **списке параметров** в связи — приёмник грузится **порциями по 1000** записей; без списка — **по одной** строке (медленно).
5. Одинаковые имена полей в связанных наборах → значение из **родителя**.
6. Зависимый набор не грузится без родителя (если нет общего предка с тем же полем).
7. Неиспользуемые в настройках поля приёмника → набор может не войти в макет.
8. В одной группировке нельзя смешивать поля **несвязанных** наборов (итоги — исключение).
9. При нескольких наборах в группировке обход идёт по **последнему зависимому** набору.

## Связь по периодам

Для остатков по месяцам + продажи за месяц — два набора с полем периода и специальной связью «по периодам» (пример 10, стр. 112). Набор остатков часто на `ОстаткиИОбороты` с полем `Период`.

См. `05-parameters-periods.md`.
