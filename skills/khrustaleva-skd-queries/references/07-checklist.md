# Чеклист: запрос 1С и отчёт СКД

На основе Хрусталева (изд. 2). Перед сдачей запроса в коде, консоли или отчёта.

## Язык запросов (модули, консоль, любой запрос)

- [ ] Виртуальные таблицы: отбор в **параметрах** ВТ
- [ ] LEFT JOIN: нужен полный левый набор — порядок таблиц и тип соединения
- [ ] Псевдонимы таблиц и полей понятны
- [ ] Период и ссылки — через `&Параметры`, не склейка строк
- [ ] `В (&Ссылка)` / `В (&Массив)` для ограничения дочерней выборки
- [ ] При `Регистратор` из `ОстаткиИОбороты` — поля периода (см. `08`, `06`)

Подробно: `08-query-language-general.md`.

## Архитектура (СКД)

- [ ] Выбрана модель: **один запрос** / **связи наборов** / **объединение** (см. `04-union-vs-link-vs-query.md`)
- [ ] Не использовано «Объединение» там, где нужна связь по ключу (и наоборот)
- [ ] Тяжёлый JOIN разбит на наборы, если запрос нечитаем или дублирует строки

## Запрос

- [ ] Псевдонимы полей и таблиц понятны (`спрНоменклатура`, не `Т`)
- [ ] LEFT JOIN: флаг «Все» на **левой** таблице, если нужны все элементы справочника
- [ ] Отсекающий отбор — в `ГДЕ` запроса; гибкая сортировка — в настройках СКД
- [ ] Виртуальные таблицы: условия в **параметрах** ВТ, не только в `ГДЕ` после соединения
- [ ] Для связанных наборов: `В (&ПараметрСвязи)` в запросе **приёмника**

## Связи наборов

- [ ] Определён **родитель** (чьи строки «главные»)
- [ ] Учтено: глобальный отбор на приёмнике → **INNER** вместо LEFT
- [ ] Нет одинаковых имён полей в несвязанных наборах
- [ ] В группировке нет полей из несвязанных наборов

## Параметры

- [ ] Период задан в запросе, если только объектные таблицы
- [ ] `КонецПериода` с `КонецПериода(&КонецПериода, "День")` при необходимости
- [ ] Обязательные параметры: **Использование = Всегда**

## Остатки и итоги

- [ ] Полям остатков назначены роли **Остатки** / периоды / измерения
- [ ] При детализации по `Регистратор` выбраны родительские **Период*** поля
- [ ] Нумерация периодов непрерывна, уточняющие периоды имеют меньший номер

## Производительность

- [ ] Параметр связи ограничивает дочерний набор
- [ ] Для списков в связи разрешён режим **порций по 1000**, если применимо
- [ ] Нет выборки «всех документов» с отбрасыванием при компоновке
